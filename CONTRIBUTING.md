## How Can I Contribute?

### 1. Reporting Bugs
*   Check the [XDB Issues List](https://github.com/xdb-project/xdb/issues) to see if the bug has already been reported.
*   If not, open a [New Bug Report](https://github.com/xdb-project/xdb/issues/new?labels=bug). Please provide as much context as possible to help us reproduce the issue.

### 2. Suggesting Enhancements
*   Open an [Enhancement Request](https://github.com/xdb-project/xdb/issues/new?labels=enhancement) on our issues page.
*   Explain why this feature would be valuable for the [XDB-Project](https://github.com/xdb-project/xdb).

### 3. Pull Requests (PRs)
Ready to submit a change? Follow these steps:
1.  **Fork** the repository directly here: [Fork xdb-project/xdb](https://github.com/xdb-project/xdb/fork).
2.  **Create a branch** for your feature or bug fix: `git checkout -b feature/your-feature-name`.
3.  **Commit your changes** with clear and descriptive messages.
4.  **Push to your fork** and **submit a [Pull Request](https://github.com/xdb-project/xdb/pulls)** to our `main` branch.

## Development Setup

To start developing locally, use ini commands:

1.  **Clone your fork**: 
    ```bash
    git clone https://github.com/xdb-project/xdb.git
    ```
2.  **Add upstream** to keep your local code synced with the main project:
    ```bash
    git remote add upstream https://github.com/xdb-project/xdb.git
    ```
3.  **Update your local main branch**:
    ```bash
    git pull upstream main
    ```

## Style Guide
*   **Consistency**: Follow the coding style and run `make format` before committing. [XDB Source Code](https://github.com/xdb-project/xdb).
*   **Documentation**: If you add a new feature, please update the [README.md](https://github.com/xdb-project/xdb/blob/main/README.md).
*   **Atomic Commits**: Keep your PRs focused. Submit separate PRs for unrelated fixes.

## Recognition
All contributors who have their PRs merged will be officially recognized in our [CONTRIBUTORS.md](https://github.com/xdb-project/xdb/blob/main/CONTRIBUTORS.md) file.
